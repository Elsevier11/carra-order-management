# ADR-0001 — Uso di Markdown su Git/GitHub come base documentale

## Stato
Accepted

## Contesto

Il progetto richiede continuità nel tempo, versionamento, facile modifica dei documenti e possibilità di collegare requisiti, modello dati, workflow e backlog.

## Decisione

Usare un repository Git/GitHub con documentazione principale in file Markdown organizzati per cartelle.

## Conseguenze

- Versionamento nativo delle modifiche
- Facile review e confronto cambiamenti
- Bassa dipendenza da tool proprietari
- Possibilità futura di pubblicazione documentazione con strumenti statici
